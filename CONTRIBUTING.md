# Contributing to Koala Devs 🐨

First off, thank you for considering contributing to Koala Devs! We are building a community where anyone can learn and grow. Following these guidelines helps us process your contributions faster.

## ✅ The "Dos" (Best Practices)
* **Do** check if an issue already exists before opening a new one to avoid duplicates.
* **Do** use descriptive branch names like `feature/add-login` or `bugfix/fix-header-css`.
* **Do** keep your Pull Requests (PRs) focused on a single change; smaller PRs are easier to review and merge.
* **Do** write clear, concise commit messages (e.g., "Fix: resolve timeout error in bot.py").
* **Do** include screenshots or GIFs for any UI-related changes so we can see the impact.

## ❌ The "Don'ts" (Avoid These)
* **Don't** submit a Pull Request without an associated issue unless it’s a very minor typo fix.
* **Don't** push changes directly to the `main` branch; always use a fork or a feature branch.
* **Don't** include sensitive information like API keys, bot tokens, or personal passwords in your code.
* **Don't** ignore the GitHub Action results—if the "Python Quality Check" fails, please fix the errors before asking for a review.
* **Don't** be afraid to ask questions! We’d rather help you get it right than have you struggle in silence.

## 🚀 Getting Started
1. **Find an Issue:** Look for issues labeled `good first issue` if you are new.
2. **Fork & Clone:** Create your own copy of the repository.
3. **Set Up Environment:** Ensure you have the right version of Python or Node.js installed.
4. **Test Your Code:** Run your scripts locally before pushing to GitHub.
5. **Open a PR:** Once you're ready, submit your changes for review!

## ⚖️ Code Style
* Please follow **PEP 8** for Python projects.
* Use 2-space or 4-space indentation consistently based on the project’s existing files.
* Comment your code where the logic is complex so others can follow along.
